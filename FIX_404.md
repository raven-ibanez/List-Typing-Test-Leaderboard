# Fixing 404 Error on Vercel

If you're getting a 404 error, it's likely because Vercel isn't finding your files. Here are the solutions:

## Solution 1: Update vercel.json (Already Done)

I've updated `vercel.json` to properly route requests. Make sure you've committed and pushed this file.

## Solution 2: Check Your File Structure

Make sure your project has this structure:
```
/
├── api/
│   ├── admin/
│   ├── rank/
│   ├── leaderboard.js
│   ├── test.js
│   └── utils.js
├── public/
│   ├── index.html
│   ├── admin.html
│   ├── leaderboard.js
│   ├── admin.js
│   └── styles.css
├── package.json
├── vercel.json
└── (other files)
```

## Solution 3: Alternative - Move Public Files to Root

If the 404 persists, you can move files from `/public` to the root:

1. Move `public/index.html` → `index.html`
2. Move `public/admin.html` → `admin.html`
3. Move `public/*.js` → `*.js` (to root)
4. Move `public/*.css` → `*.css` (to root)
5. Update `vercel.json` to remove the public rewrite

But try Solution 1 first!

## Solution 4: Check Vercel Project Settings

1. Go to Vercel Dashboard → Your Project → Settings
2. Check "Root Directory" - it should be empty (or `.`)
3. Check "Build Command" - should be empty or auto-detected
4. Check "Output Directory" - should be empty

## Testing After Fix

After pushing the updated `vercel.json`:

1. **Test the homepage:**
   - Visit: `https://your-project.vercel.app/`
   - Should show the leaderboard page

2. **Test API:**
   - Visit: `https://your-project.vercel.app/api/test`
   - Should return JSON

3. **Test admin:**
   - Visit: `https://your-project.vercel.app/admin.html`
   - Should show admin login page

## If Still Getting 404

1. **Check Vercel Build Logs:**
   - Go to Deployments → Click latest deployment → View logs
   - Look for any errors

2. **Verify Files Are Committed:**
   ```bash
   git status
   git add .
   git commit -m "Fix 404 routing"
   git push
   ```

3. **Redeploy:**
   - Vercel Dashboard → Deployments → Three dots → Redeploy

4. **Check Function Logs:**
   - Functions tab → Click a function → Check logs for errors

## Common Issues

- **Files not in Git:** Make sure all files are committed
- **Wrong directory:** Check that files are in the right place
- **Cache issues:** Clear Vercel build cache and redeploy
- **Node version:** Set to 18.x or 20.x in settings
